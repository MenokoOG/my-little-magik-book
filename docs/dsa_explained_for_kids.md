# Data Structures & Algorithms Explained for Kids

Imagine you have a giant box of trading cards.

## Arrays (a list)

An array is like a row of card slots. You can put cards in order and quickly grab one by position.

## Hash maps (a magic dictionary)

A hash map is like a notebook where you write:

- card id -> card details

It helps you find things super fast.

## Stacks (last in, first out)

Like a stack of plates. The last plate you put on top is the first one you take off.

## Queues (first in, first out)

Like waiting in line for a ride. First person in line goes first.

## Graphs (friends network)

A graph is dots and lines:

- dot = person
- line = friendship

This helps apps find mutual friends or friend suggestions.

## Sorting

Sorting is arranging cards by mana cost, name, or rarity.

## Searching

Searching means finding one card among many cards quickly.

Great software uses the right data structure so things feel fast and fun.
